SpringBoot编写单元测试

发表于 2022-02-17 分类于 SpringBoot 阅读次数：

单元测试不仅可以增强程序的健壮性，而且也为程序的重构提供了依据。在实际项目开发中，单元测试与业务代码通常都会要求同步进行。有一个术语：**测试驱动开发(TDD)**，即为先编写单元测试，然后努力去开发业务代码去满足所有的单元测试用例。

测试RESTful 接口

Spring Boot提供了**@SpringBootTest注解，该注解用于修饰单元测试用例类。测试用例的测试方法依然使用@Test或@ParameterizedTest注解修饰**。@Test是普通测试，@ParameterizedTest注解是JUnit 5.x新增的测试注解，用于表示参数化测试。

测试用例类使用@SpringBootTest(webEnvironment = WebEnvironment.RANDOM_PORT)修饰，这表示在运行测试时，将会为web服务器随机分配端口，因此不需要知道Web服务器的端口是多少，就可以直接进行测试；如果想使用固定的端口，则可将webEnvironment属性指定为WebEnvironment.DEFINED_PORT，这样Spring Boot就会读取项目配置文件（如 application.properties）中的端口(server.port属性)来启动Web服务器，若没有配port属性的话，默认值为8080端口。

测试用例类会接收容器依赖注入TestRestTemplate对象，这个对象实际是对 RestTemplate进行了封装，可以在测试环境中更方便地使用 RestTemplate的功能，因此TestRestTemplate主要用于测试RESTful接口的功能， 测试方法可通过TestRestTemplate来调用RESTful接口的方法。

为此在controller层增加几个用于测试的方法，然后在test包下编写相关测试类，注意：测试用例应该定义在和被测试类位于同一个包中。

模拟Web环境测试控制器

在设置@SpringBootTest 的webEnvironment属性时，不管是设置为RANDOM_PORT，还是置为DEFINED PORT，在运行单元测试时，都会启动一个真实的 Web服务器。如果不想启动真实的 Web 服务器，则可以将webEnvironment属性设置为 WebEnvironment.MOCK，该属性值设置启动模拟的 Web服务器。即为@SpringBootTest(webEnvironment=WebEnvironment.MOCK)。

@SpringBootTest(webEnvironment=WebEnvironment.MOCK)和**@AutoConfigureMockMvc修饰测试用例类， 测试用例类定义接受依赖注入的MockMvc类型的实例变量**，通过该实例变量可用于模拟浏览器来发送请求、获取返回值：ModelAndView。

MockMvc的作用类似于测试RESTful API的TestRestTemplate，可用于模拟浏览器发送等各种请求。

使用MockMvc执行测试的方法的步骤：

  • 使用 MockMvcRequestBuilders的get()、post()、put()、patch()、delete()、options()、head()等
    法创建对应的请求；如果需要设置请求参数、请求头等，则接着调用 MockHttpServletRequestBuile的param()、header()等方法。
  • 调用MockMvc对象的perform()方法执行请求。MockMvc 的perform()方法返回 ResultActions，通过该对象的返回值可读取到控制器处理法的 ModelAndView，还可通过getResponse()获取控制器处理方法返回的响应，具体读取哪种息根据测试需求决定。

测试业务组件

Service组件其实就是一个普通的组件，它不需要借助于Web服务器，也不需要启动模拟Web环境；测试Service组件无需启动Web服务器，故使用**@SpringBootTest(webEnvironment = WebEnvironment.NONE)修饰测试用例类即可，测试用例类定义接受依赖注入的Service类型的实例变量，然后通过该变量测试Service组件的方法**。

使用模拟组件

假设要测试的A组件依赖于B组件，但B组件可能根本就没有开发完成，或B组件开发出来了，但没有经过严格的测试，这样如果直接对A组件进行测试，此时可能测试出现的错误并不是A组件导致的，而是A组件所依赖的B组件所导致的。为了避免B组件将错误传给A组件或者在没有B组件的前提下，能对A组件进行测试，此时就需要提供一个B组件的模拟对象——借助于Mockit即可。

步骤：

  • 定义一个**@MockBean修饰的模拟组件，该模拟组件就代表了被依赖组件**（该组件可能未开发完成，或未经过严格测试）
  • 在测试方法中，调用BDDMockito的given()类方法为模拟组件的指定方法直接设置返回值，这样即可基于对模拟组件所设置的返回值来进行测试。
